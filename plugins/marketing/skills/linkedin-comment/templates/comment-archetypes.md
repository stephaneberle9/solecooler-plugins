# Comment Archetypes — fill-in structures

Six structures for comments that earn reactions and rank as "Top comment." Each is built so the **value lands in the first 1–2 lines** (before LinkedIn's "…more" cutoff). Fill the brackets with specifics from the original post and the user's context — don't paraphrase the structure into something "loosely similar." The shapes are what make them land.

Use these *after* picking an angle, not as a menu to dump on the user. Pick the structure that fits the chosen angle, fill it, and tighten it to the user's voice.

---

## 1. The new fact

Adds a concrete data point, source, or detail the post didn't have. The single highest-reach archetype, because it gives readers something they didn't already know.

```
[Concrete fact / number / source that extends the post] —
[one line on why it matters for the author's point].
[Optional: source or link, at the very end.]
```

**Example**
```
There's a 2024 BGH ruling on exactly this case — it flipped the burden of proof onto the platform.
Changes the whole calculus for anyone relying on the old guidance.
```

---

## 2. The sharp counter

A respectful, specific disagreement or nuance. Never "I disagree" — always *why*, in one line, up front.

```
[The specific thing that's not quite right] — [the sharper version], because [one concrete reason].
```

**Example**
```
The 10x framing undersells it. The real unlock isn't speed, it's that you can now try ideas you'd never have staffed before.
```

---

## 3. The build-on

Takes the author's point one step further than they did. Makes you look like you saw what they missed.

```
[Restate their point in 4–6 words]. The part people miss: [the next-step insight they didn't reach].
```

**Example**
```
Great point on async. The part people miss: it only works if you also kill the "quick sync" reflex — otherwise you've just added a layer.
```

---

## 4. The lived experience

A short, concrete first-hand result. Specific numbers and details beat general agreement.

```
[Did the exact thing the post describes]. [Concrete result / what actually happened].
[Optional: the one thing that surprised you.]
```

**Example**
```
Ran exactly this for six months across 40 reps. The pipeline didn't grow — but close rate jumped 18%, which nobody predicted.
```

---

## 5. The reframe

Restate the post's core idea in a sharper, more memorable line — the kind an author wants to pin.

```
[The post's idea, compressed into one quotable line].
```

**Example**
```
Documentation isn't writing things down. It's deciding what future-you is allowed to forget.
```

---

## 6. The precise question

A question the author and other readers actually want to answer. Drives replies — the strongest "Top comment" signal.

```
[Specific, answerable question that opens up the post's idea] — [optional: the assumption you're testing].
```

**Example**
```
Curious where you draw the line on this — does it still hold when the team is fully remote, or does the trust cost flip it?
```

---

## Filling rules

- **Value in line 1.** If the payload isn't in the first sentence, restructure.
- **One idea per comment.** Don't stack two archetypes — pick the strongest.
- **No leftover brackets.** Every comment ships ready to paste.
- **Link last, optional, one max.** See [references/comment-craft.md](../references/comment-craft.md) section 4.
- **Sound like the user.** Tighten every filled structure to their voice before delivering.
