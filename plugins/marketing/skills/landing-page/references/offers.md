# Offers — Introduction, Components, and Testimonials

Step 3 in writing order. Section 4 on the page. Load when it's time to build the offer section.

---

## Offer structure — 4 parts

### Part A — Introduce it

State the offer name. Show a quality image/mockup if available. One sentence describing what it is.

### Part B — The big promise

Short paragraph or tight list covering:

- The main promise / outcome
- What makes it different (differentiation)
- Timeframe to results
- The biggest objection handled
- Why it's easy / quick
- Proof

### Part C — The specifics (components)

Break the offer into named components. Each component:

- Has a name (and ideally an image)
- Has a one-sentence purpose
- Lists 3–9 specific benefit bullets

Format:

```
[Component Name]

[One sentence describing what this delivers]

What's inside:
- [Specific benefit 1]
- [Specific benefit 2]
- [Specific benefit 3]
- [Specific benefit 4]
```

### Part D — Testimonials

3–5 testimonials. Not a wall of 20. Each one:

- Best phrase pulled out as a headline above the quote
- Full name + location or title
- Photo if the user has it
- 2–4 sentence testimonial body

## Offer section gotchas

1. **One big paragraph describing the offer.** Nobody reads it. Break into named components with bullets.

2. **Components without specific benefits.** "Module 2: Leads" with a vague paragraph is not a component. 3–9 specific bullets per component.

3. **Wall of testimonials.** Five great testimonials beat twenty mediocre ones. Prune ruthlessly.

4. **Vague testimonials.** "Really helped me" does nothing. Specific numbers, specific timeframes, specific outcomes.

5. **Fabricated testimonials.** Don't write them. Mark `[TESTIMONIAL PLACEHOLDER]` and flag it in the output for the user to fill in.

6. **No bonuses.** Bonuses handle objections and raise perceived value. At minimum, include one that makes implementation easier (template pack, checklist, kit).

7. **Bonuses that feel like padding.** A bonus should have a reason to exist ("for people worried about X, this solves it"). A bonus that's just "extra value" reads like filler.
